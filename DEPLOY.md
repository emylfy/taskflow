# Деплой TaskFlow на Timeweb Cloud

Инструкция по размещению проекта на облачном сервере **Timeweb Cloud** с
доменом в зоне `.ru` и автоматическим HTTPS (сертификат Let's Encrypt
выпускает Caddy). Все данные хранятся на сервере в РФ — это закрывает
требование 152-ФЗ о локализации персональных данных.

Стек: Docker Compose (Next.js + кастомный Node-сервер с WebSocket →
PostgreSQL → Caddy). Поэтому нужен **облачный сервер (VPS)**, обычный
shared-хостинг не подойдёт.

WebSocket совместного редактирования и чата работает «из коробки»: collab-сервер
поднимается на отдельном порту (PORT+1), а маршрут `/api/collaboration` уже прописан
в `Caddyfile` и проксируется на него. Отдельной ручной настройки WS и переменной
`NEXT_PUBLIC_COLLAB_URL` не требуется (последняя нужна, только если выносить WS на
отдельный хост/домен).

---

## 0. Что понадобится

- Аккаунт на [Timeweb Cloud](https://timeweb.cloud).
- Домен `.ru` (для регистрации нужны паспортные данные — таково правило зоны).
- 20 минут времени.

---

## 1. Регистрация домена

Панель Timeweb → **Домены** → зарегистрировать домен в зоне `.ru`
(~200–300 ₽/год). Если домен уже куплен у другого регистратора — можно
оставить там, но тогда DNS-записи (шаг 3) добавляются в его панели.

---

## 2. Создание облачного сервера

Панель Timeweb → **Облачные серверы** → *Создать*:

- **ОС:** Ubuntu 24.04.
- **Конфигурация:** минимум **2 vCPU / 4 ГБ RAM**. Сборка `next build`
  идёт прямо на сервере и съедает >1.5 ГБ — на 1 ГБ упадёт по нехватке
  памяти. Если взял тариф поменьше — обязательно сделай swap (шаг 4).
- **Регион:** Россия (по умолчанию) — нужно для 152-ФЗ.

После создания запиши **публичный IP** сервера и пароль root (или подключи
свой SSH-ключ).

---

## 3. DNS: домен → IP сервера

Панель Timeweb → **Домены** → твой домен → *DNS-записи*. Добавь две записи
типа **A** на IP сервера:

| Тип | Поддомен | Значение (IP) |
|-----|----------|---------------|
| A   | `@`      | IP сервера    |
| A   | `www`    | IP сервера    |

Две записи нужны, потому что Caddy обслуживает и `домен`, и `www.домен`.
Распространение DNS — от нескольких минут до часа. Проверить:

```bash
nslookup твой-домен.ru
```

Должен вернуться IP сервера. **Не запускай Caddy, пока домен не резолвится** —
иначе выпуск сертификата не пройдёт (Let's Encrypt проверяет домен по HTTP).

---

## 4. Базовая настройка сервера

Подключись по SSH (под root):

```bash
ssh root@IP-сервера
```

Обнови систему и поставь файрвол:

```bash
apt update && apt upgrade -y

# Файрвол: только SSH + HTTP + HTTPS. Порт БД 5432 наружу НЕ открываем.
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
```

**Swap** (обязательно для сервера с 2 ГБ RAM, чтобы сборка не падала):

```bash
fallocate -l 4G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

---

## 5. Установка Docker

```bash
curl -fsSL https://get.docker.com | sh
```

Скрипт ставит Docker Engine вместе с плагином `docker compose`. Проверка:

```bash
docker --version && docker compose version
```

## 6. Код и переменные окружения

```bash
cd /opt
git clone https://github.com/emylfy/taskflow-diploma.git taskflow
cd taskflow
cp .env.example .env
nano .env
```

Заполни `.env` (это файл с секретами, в git он не попадает):

| Переменная | Значение |
|------------|----------|
| `APP_DOMAIN` | твой домен **без** `https://`, напр. `taskflow.ru` |
| `BETTER_AUTH_URL` | `https://твой-домен.ru` (тот же домен) |
| `BETTER_AUTH_SECRET` | сгенерируй: `openssl rand -base64 32` |
| `POSTGRES_PASSWORD` | надёжный пароль |
| `DATABASE_URL` | `postgresql://taskflow:ТОТ_ЖЕ_ПАРОЛЬ@db:5432/taskflow` (хост `db` — это имя сервиса в compose, не меняй) |
| `DEMO_MODE` | `true` для демо на защите (быстрый вход без почты), `false` для боевого режима |

Опционально, если будешь показывать вход по почте / оплату:
`SMTP_*`, `YANDEX_CLIENT_*`, `YOOKASSA_*`. Без них приложение работает
(magic-link печатается в лог, Яндекс-вход и оплата просто отключены).

> Пароль в `POSTGRES_PASSWORD` и пароль внутри `DATABASE_URL` должны
> совпадать — иначе приложение не подключится к базе.

---

## 7. Запуск

```bash
docker compose up -d --build
```

Первая сборка идёт несколько минут. Следи за выпуском сертификата:

```bash
docker compose logs -f caddy
```

Когда увидишь `certificate obtained successfully` — HTTPS готов. `Ctrl+C`
выходит из логов (контейнеры продолжают работать).

## Чеклист перед защитой

- [ ] DNS A-записи `@` и `www` указывают на IP сервера.
- [ ] `ufw` включён, 5432 наружу не торчит.
- [ ] `BETTER_AUTH_SECRET` и `POSTGRES_PASSWORD` — не из примера, свои.
- [ ] HTTPS работает, сертификат валиден.
- [ ] Решено, что с `DEMO_MODE`: для демонстрации удобнее `true`.

---

## Если что-то не так

| Симптом | Что проверить |
|---------|---------------|
| Caddy не выпускает сертификат | Резолвится ли домен на IP (`nslookup`); открыты ли порты 80/443; логи `docker compose logs caddy` |
| Сборка падает / сервер зависает | Не хватает RAM — сделай swap (шаг 4) |
| Сайт отдаёт 502 | Приложение не поднялось: `docker compose logs app` |
| `APP_DOMAIN variable is not set` | Не заполнен `APP_DOMAIN` в `.env` |
| Не подключается к БД | Пароль в `DATABASE_URL` ≠ `POSTGRES_PASSWORD`, либо хост не `db` |
